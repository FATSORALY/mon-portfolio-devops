Comment j'ai déployé mon premier cluster Kubernetes from scratch

📅 15 Mars 2026 • ⏱️ 8 min de lecture

Kubernetes est devenu la référence en matière d'orchestration de conteneurs. Dans cet article, je vais vous partager mon expérience de déploiement d'un premier cluster Kubernetes sur des serveurs Ubuntu.

Prérequis nécessaires

Avant de commencer, assurez-vous d'avoir :

  • Au moins 2 machines Ubuntu 22.04 (1 master + 1 worker)
  • 2GB de RAM minimum par machine
  • Connexion internet pour télécharger les images Docker

Installation de containerd et Kubernetes

# Sur toutes les machines
sudo apt-get update
sudo apt-get install -y containerd

# Configurer containerd
sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml
sudo systemctl restart containerd

# Désactiver le swap
sudo swapoff -a
sudo sed -i '/ swap / s/^/#/' /etc/fstab

# Installer kubeadm, kubelet, kubectl
sudo apt-get install -y apt-transport-https ca-certificates curl
curl -fsSL https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
echo "deb https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

Initialisation du cluster (sur le master)

sudo kubeadm init --pod-network-cidr=10.244.0.0/16

# Configurer kubectl pour l'utilisateur courant
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

# Installer Flannel (CNI)
kubectl apply -f https://raw.githubusercontent.com/flannel-io/flannel/master/Documentation/kube-flannel.yml

Ajouter des workers au cluster

Sur le master, récupérez la commande d'join :

kubeadm token create --print-join-command

Exécutez cette commande sur chaque machine worker.

Vérification du cluster

kubectl get nodes
kubectl get pods --all-namespaces

Déploiement d'une première application test

kubectl create deployment nginx --image=nginx
kubectl expose deployment nginx --port=80 --type=NodePort
kubectl get svc

Félicitations ! Vous avez maintenant un cluster Kubernetes fonctionnel. La prochaine étape sera d'ajouter un stockage persistant et un ingress controller.

Prochaines étapes recommandées

  • Installer un Ingress Controller (NGINX ou Traefik)
  • Mettre en place cert-manager pour les certificats SSL
  • Configurer Prometheus + Grafana pour le monitoring
  • Utiliser Helm pour gérer les déploiements
