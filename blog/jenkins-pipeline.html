Jenkins : Maîtrisez les pipelines CI/CD comme un pro

📅 25 Mars 2026 • ⏱️ 15 min de lecture

Jenkins est l'outil d'intégration continue le plus populaire au monde. Dans cet article, je vous guide pas à pas pour installer, configurer et créer des pipelines Jenkins professionnels, du job simple à la pipeline complète avec Docker et Kubernetes.

Pourquoi Jenkins encore en 2026 ?

Malgré l'arrivée de GitLab CI, GitHub Actions ou CircleCI, Jenkins reste incontournable car :

  • ✅ Gratuit et open-source (aucune limite de jobs ou de runners)
  • ✅ Extrêmement personnalisable (plus de 1800 plugins)
  • ✅ On-premise : vos données restent chez vous
  • ✅ Communauté immense : solutions à tous les problèmes

Installation de Jenkins

Méthode 1 : Avec Docker (recommandée)

# Créer un volume persistant
docker volume create jenkins-data

# Lancer Jenkins
docker run -d \
  --name jenkins \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkins/jenkins:lts

# Récupérer le mot de passe initial
docker logs jenkins | grep -A 5 "Please use the following password"

Méthode 2 : Installation directe (Ubuntu/Debian)

# Ajouter le dépôt Jenkins
curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key | sudo tee /usr/share/keyrings/jenkins-keyring.asc > /dev/null
echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] https://pkg.jenkins.io/debian-stable binary/ | sudo tee /etc/apt/sources.list.d/jenkins.list > /dev/null

# Installer Jenkins
sudo apt update
sudo apt install jenkins -y
sudo systemctl start jenkins
sudo systemctl enable jenkins

# Accéder à http://localhost:8080
sudo cat /var/lib/jenkins/secrets/initialAdminPassword

Type 1 : Pipeline Déclarative (recommandée)

// Jenkinsfile
pipeline {
    agent any
    
    tools {
        maven 'Maven-3.9'
        jdk 'JDK-17'
    }
    
    environment {
        DOCKER_REGISTRY = 'monregistry.com'
        DOCKER_IMAGE = "${DOCKER_REGISTRY}/${JOB_NAME}:${BUILD_NUMBER}"
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Build') {
            steps {
                sh 'mvn clean compile'
            }
        }
        
        stage('Test') {
            steps {
                sh 'mvn test'
            }
            post {
                always {
                    junit 'target/surefire-reports/*.xml'
                }
            }
        }
        
        stage('SonarQube Analysis') {
            steps {
                withSonarQubeEnv('SonarQube') {
                    sh 'mvn sonar:sonar'
                }
            }
        }
        
        stage('Build Docker Image') {
            steps {
                script {
                    docker.build("${DOCKER_IMAGE}")
                }
            }
        }
        
        stage('Push Docker Image') {
            steps {
                script {
                    docker.withRegistry("https://${DOCKER_REGISTRY}", 'docker-credentials') {
                        docker.image("${DOCKER_IMAGE}").push()
                    }
                }
            }
        }
        
        stage('Deploy to Staging') {
            steps {
                sh 'kubectl set image deployment/monapp monapp=${DOCKER_IMAGE} -n staging'
            }
        }
        
        stage('Integration Tests') {
            steps {
                sh 'npm run test:integration'
            }
        }
    }
    
    post {
        success {
            emailext (
                subject: "Pipeline réussi : ${JOB_NAME} - ${BUILD_NUMBER}",
                body: "Le build ${BUILD_NUMBER} a été déployé avec succès.",
                to: 'team@monentreprise.com'
            )
        }
        failure {
            emailext (
                subject: "Pipeline échoué : ${JOB_NAME} - ${BUILD_NUMBER}",
                body: "Le build ${BUILD_NUMBER} a échoué. Vérifiez les logs.",
                to: 'team@monentreprise.com'
            )
        }
    }
}

Type 2 : Pipeline Scriptée (plus flexible)

// Jenkinsfile scripté
node('docker') {
    try {
        stage('Checkout') {
            checkout scm
        }
        
        stage('Build & Test') {
            sh '''
                docker build -t monapp:latest .
                docker run --rm monapp:latest npm test
            '''
        }
        
        stage('Security Scan') {
            sh 'trivy image monapp:latest'
        }
        
        stage('Push to Registry') {
            withCredentials([string(credentialsId: 'docker-hub', variable: 'DOCKER_PASSWORD')]) {
                sh '''
                    docker tag monapp:latest monuser/monapp:${BUILD_NUMBER}
                    docker login -u monuser -p ${DOCKER_PASSWORD}
                    docker push monuser/monapp:${BUILD_NUMBER}
                '''
            }
        }
        
        stage('Deploy') {
            input message: 'Déployer en production ?', ok: 'Oui'
            sh 'kubectl rollout status deployment/monapp'
        }
        
    } catch (e) {
        currentBuild.result = 'FAILURE'
        throw e
    } finally {
        stage('Cleanup') {
            sh 'docker system prune -f'
        }
    }
}

Pipeline multi-branche avec Jenkinsfile

// Jenkinsfile pour gestion multi-environnements
pipeline {
    agent any
    
    stages {
        stage('Build') {
            steps {
                sh 'npm ci'
                sh 'npm run build'
            }
        }
        
        stage('Test') {
            parallel {
                stage('Unit Tests') {
                    steps { sh 'npm run test:unit' }
                }
                stage('Lint') {
                    steps { sh 'npm run lint' }
                }
                stage('Security') {
                    steps { sh 'npm audit' }
                }
            }
        }
        
        stage('Deploy') {
            steps {
                script {
                    switch(env.BRANCH_NAME) {
                        case 'main':
                            sh 'kubectl apply -f k8s/production/'
                            break
                        case 'develop':
                            sh 'kubectl apply -f k8s/staging/'
                            break
                        default:
                            sh 'echo "Pas de déploiement pour cette branche"'
                    }
                }
            }
        }
    }
}

Configuration des agents (workers) Jenkins

Agent Docker

pipeline {
    agent {
        docker {
            image 'node:18-alpine'
            args '-v /tmp:/tmp'
        }
    }
    stages {
        stage('Build') {
            steps {
                sh 'npm install'
                sh 'npm run build'
            }
        }
    }
}

Agent Kubernetes (pods éphémères)

pipeline {
    agent {
        kubernetes {
            yaml '''
apiVersion: v1
kind: Pod
spec:
  containers:
  - name: docker
    image: docker:24-dind
    command: ["dockerd-entrypoint.sh"]
    securityContext:
      privileged: true
  - name: kubectl
    image: bitnami/kubectl:latest
    command: ["cat"]
'''
        }
    }
    stages {
        stage('Build & Deploy') {
            steps {
                container('docker') {
                    sh 'docker build -t monapp .'
                }
                container('kubectl') {
                    sh 'kubectl apply -f deployment.yaml'
                }
            }
        }
    }
}

Bonnes pratiques Jenkins

  • ✅ Utiliser Jenkinsfile : versionner vos pipelines (Pipeline as Code)
  • ✅ Séparer les stages : Build, Test, Deploy bien distincts
  • ✅ Nettoyer les workspaces : éviter de saturer le disque
  • ✅ Utiliser des credentials : jamais de secrets en dur
  • ✅ Backup régulier : sauvegarder /var/jenkins_home
  • ✅ Limiter les droits : Matrix Authorization Strategy

Monitoring des pipelines

// Notifications Slack
post {
    success {
        slackSend(
            color: 'good',
            message: "Pipeline réussi : ${JOB_NAME} - ${BUILD_NUMBER}",
            channel: '#ci-cd'
        )
    }
    failure {
        slackSend(
            color: 'danger',
            message: "Pipeline échoué : ${JOB_NAME} - ${BUILD_NUMBER}\nLogs: ${BUILD_URL}",
            channel: '#ci-cd'
        )
    }
}

Dépannage des erreurs courantes

ErreurSolution
Permission denied (Docker)Ajouter jenkins au groupe docker : sudo usermod -aG docker jenkins
Out of memoryAugmenter la RAM : JAVA_OPTS="-Xmx2048m"
Pipeline stuckVérifier les agents disponibles : Manage Jenkins → Nodes
Plugin incompatibilityUtiliser les versions LTS des plugins

Alternative : Jenkins avec Docker Compose

# docker-compose.yml
version: '3.8'
services:
  jenkins:
    image: jenkins/jenkins:lts
    container_name: jenkins
    ports:
      - "8080:8080"
      - "50000:50000"
    volumes:
      - jenkins-data:/var/jenkins_home
      - /var/run/docker.sock:/var/run/docker.sock

      - JAVA_OPTS=-Djenkins.install.runSetupWizard=false

    image: jenkins/ssh-agent:latest
